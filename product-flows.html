Gap Analysis v2 · Audit

10 Known Flaws — Prioritized

Post-P0/P1/P2 architecture audit. Entitlements, code-splitting, and tests are now resolved. These are the open flaws remaining as of v4.0.

P0 · Blockers (Sprint 6)

  • F-01 · localStorage-only (no cloud)
  • F-02 · Demo paywall, no real billing
  • F-10 · No real auth (UI-only SSO)

P1 · High Value (Sprints 3–4)

  • F-03 · No auto-pipeline on schedule
  • F-04 · 6 hollow stub views
  • F-05 · AI quota UX missing
  • F-06 · CommandCenter no daily brief

P2 · Polish (Sprint 5)

  • F-07 · No error/retry UI in AI calls

  • F-09 · Sidebar quota badge missing

  • ✓ Lazy-loading all 40+ views
F-01 · localStorage-Only Persistence P0 · Sprint 4

Problem: All creator data (ideas, scripts, analytics) lives in Zustand + localStorage. One browser wipe = permanent data loss.
Impact: No cross-device access, no team collab, no backup. Blocks paying users.
Fix: Dexie.js IndexedDB adapter + Supabase cloud sync in Sprint 4.

F-02 · Demo Paywall — No Real Billing P0 · Sprint 6

Problem: canAccess() and PaywallModal work correctly in UI but no payment processor is wired — upgrades are instant and free.
Impact: Zero monetization. Product cannot generate revenue.
Fix: Razorpay/Stripe integration with webhook-confirmed tier upgrades in Sprint 6.

F-10 · No Real Auth (UI-Only SSO) P0 · Sprint 6

Problem: AuthScreen has Google/Apple/Microsoft buttons but they're decorative. No session, no JWT, no user identity.
Impact: No user accounts = no multi-device, no teams, no billing identity.
Fix: Supabase Auth with Google OAuth provider in Sprint 6.

F-03 · No Auto-Pipeline on Schedule P1 · Sprint 3

Problem: Calendar events marked "publish" don't trigger any action. No publish queue, no scheduler, no pipeline push.
Impact: "Autopilot" mode is completely non-functional as a core loop feature.
Fix: useScheduler hook + ContentPipeline auto-move on publish date hit.

F-04 · 6 Hollow Stub Views P1 · Sprint 5

Problem: AutomationEngine, OpsHQ, CreatorStorefront, ContentABLab, AudienceLayer, CollabNetwork render placeholder UI with no functionality.
Impact: 15% of navigation map is dead weight; undermines perceived product quality.
Fix: Implement core interaction loop for each stub in Sprint 5.

F-05 · AI Quota UX Missing P1 · Sprint 3

Problem: TIER_LIMITS.aiGenerationsPerMonth is enforced in APIProxy but no meter, no warning toast, no upgrade nudge is shown in-app.
Impact: Users hit quota wall with no explanation; looks like a bug.
Fix: useAIQuota() hook + sidebar usage bar + toast warnings at 80%/100%.

F-06 · CommandCenter Lacks Daily Brief P1 · Sprint 3

Problem: The flagship "daily brief" card (AI-generated priorities) is missing from CommandCenter dashboard. Existing cards are static.
Impact: The #1 differentiator in onboarding copy doesn't exist yet.
Fix: DailyBrief component using aiFetchBridge with prompt built from store state.

F-07 · No Error/Retry UI in AI Calls P2 · Sprint 5

Problem: When aiFetchBridge fails (rate limit, network), UI silently shows empty results or freezes.
Impact: Feels broken; no recovery path shown to user.
Fix: Shared AIErrorState component with retry button + exponential backoff.

F-08 · TrendRadar Shows No Live Data P2 · Sprint 5

Problem: TrendRadar.tsx renders hardcoded mock trends. No API call to any trend source.
Impact: "Trend detection" feature is decorative; provides no actual signal.
Fix: Wire Claude AI prompt to synthesize trends from niche + platform context.

F-09 · Sidebar AI Quota Badge Missing P2 · Sprint 3

Problem: Sidebar has no persistent usage indicator. Quota consumption is invisible until hitting a wall.
Impact: Users can't self-manage usage; leads to frustration and churn.
Fix: Mini progress bar in sidebar footer showing X/Y AI credits used this month.
